Don't Stop 3D

The game is set in a futuristic world where the player is a robot that has become self aware, and is now trying to escape this "testing factory" of some sort.

Disclaimer

This game started out as a solo project, which I made on a school course. This spring I had the opportunity to keep working on the game, with a group to aid me.

On this current course my roles are Main programmer, Main game designer and Project lead

Features

Path Generation

At the moment the path generation is very basic, I've made some world "block" prefabs and those get spawned in a random order, which then loops. Current plan is to make more of those blocks, to easily add more to the experience.

It would be nice to have a randomly generated map with more elements that get randomized, but I think that is outside the scale of this project for this school course.

Player mechanics

Stamina

I've implemented a stamina system to the game, to make the player have something else to worry about as well, other than the map itself.

Currently basic movement such as running and jumping don't cost stamina, while sprinting and wallrunning do.

Wallrun

The current wallrun system works well, but I'm wondering whether to improve the wallrun's path. Currently the wallrun stays at a certain height, and stops when either the animation is done, or the player jumps or decides to move away from the wall.

I plan on making it seem more natural, in the sense that when the player jumps on the wall, the wallrun itself responds to the physics already affecting to the player. For example if you are already going down fast, you simply can't wallrun. Depending on how much speed the player still has going upwards, the longer the wallrun.

Sliding

While the feature is implemented in the game, the current map design doesn't have many opportunities for the feature to be used. I'm currently working on reworking the map design to fit the new movement mechanics and make it more challenging



Project Goal

ICT Showroom

The main goal is to have this project ready when we are going to present this project march 5th in ICT Showroom public event in Turku, and should have all the features mentioned above ready by then.

Learning

This is a great opportunity to learn more about the Unity's mecanim system. Blend trees and animation events are something that could prove useful, to add more elements to the player animations.
